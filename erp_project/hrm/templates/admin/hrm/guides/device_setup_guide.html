devices {% trans "Device Setup & Management Guide" %}

এই guide এ দেখানো হয়েছে কিভাবে ZK attendance device setup করবেন, employee enroll করবেন, এবং device থেকে attendance data sync করবেন।

integration_instructions Device Integration Methods

আমাদের system দুইটি method support করে: ADMS Protocol (Push) এবং TCP/IP (Pull)। আপনি যেকোনো একটি বা দুইটি একসাথে ব্যবহার করতে পারবেন।

cloud_upload Method 1: ADMS Protocol (Push-based) - Recommended

📋 ADMS কি?

ADMS (Attendance Data Management System) হলো একটি protocol যেখানে device automatically আপনার server এ data push করে। Real-time data sync এর জন্য এটি best।

Device Configuration:

Server URL: http://your-server-ip:8000/hrm/device/iclock/
Device SN: Device এর Serial Number (unique হতে হবে)
Push Interval: 1 minute (recommended)
💡 Advantage: Device automatically push করে, manual sync করার দরকার নেই। Real-time attendance tracking এর জন্য perfect।

cloud_download Method 2: TCP/IP (Pull-based)

📋 TCP/IP Method কি?

TCP/IP Method এ আপনার server device এর IP address দিয়ে connect করে এবং data fetch করে। Manual sync এর জন্য useful।

Device Configuration:

IP Address: Device এর local IP (e.g., 192.168.1.100)
Port: 4370 (default)
Password: Device admin password (if set)
💡 Use Case: ADMS configure করতে না পারলে বা manual sync করতে চাইলে এই method ব্যবহার করুন।

add_circle Step 1: Add Device to System

HRM → Device Management → ZK Devices → Add

Example Device:

Serial Number: ABCD1234567890
Name: Main Office Device
IP Address: 192.168.1.100
Port: 4370
Location: Main Office

fingerprint Step 2: Enroll Employee on Device

📋 Enrollment Process:

⚠️ Important: Device এ যে User ID দিয়ে enroll করবেন, system এ employee এর User ID ও সেই same number হতে হবে।

Enrollment Steps:

  1. Device এ Menu → User Management → New User
  2. User ID enter করুন (e.g., 001) - এটি system এর User ID এর সাথে match করতে হবে
  3. Name enter করুন
  4. Fingerprint scan করুন (minimum 2 fingers recommended)
  5. Save করুন

sync Step 3: Sync Attendance Data

ADMS Method (Automatic):

Device automatically push করবে। আপনাকে কিছু করতে হবে না। শুধু check করুন: HRM → Device Management → Device Heartbeats

TCP/IP Method (Manual):

Device list থেকে "Sync Attendance" button click করুন অথবা API call করুন: POST /hrm/device/pyzk/sync-attendance/

troubleshoot Common Issues & Solutions

❌ Device not connecting:
✓ Check network connection
✓ Verify IP address and port
✓ Check firewall settings
❌ Attendance not syncing:
✓ Check device heartbeat status
✓ Verify User ID matches in system and device
✓ Check TCP Sync Logs for errors
❌ Employee not found:
✓ Ensure employee exists in system with correct User ID
✓ Check employee status is Active
✓ Verify User ID in device matches system
